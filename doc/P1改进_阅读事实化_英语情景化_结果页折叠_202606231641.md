## 背景

来自二年级学生视角的改进计划。P1 一次性完成四项：

- #7 英语提示词情景化

## 改动详情

### 1. 阅读题改抽象为事实（`scripts/build-reading.js` → `reading.json`）

22 道成语故事题从问"道理/比喻"改为问"故事中的具体事实"，选项也改成可证伪的具体描述。

| 原题（抽象） | 新题（事实） |
|---|---|
| 「守株待兔」告诉我们什么道理？ | 农夫为什么最后没等到第二只兔子，庄稼也荒废了？ |
| 「掩耳盗铃」比喻什么？ | 偷钟的人捂住自己的耳朵是为了什么？ |
| 「指鹿为马」比喻什么？ | 赵高指着鹿对秦二世说什么？ |
| 「画蛇添足」比喻什么？ | 画蛇比赛中，第一个画完的人为什么没拿到酒？ |
| …等共 22 题 | … |

**收益**：二年级学生可以从故事文本直接定位答案，不再需要先抽象出"道理"。

### 2. 英语情景化（`scripts/build-english.js` → `english.json`）

51 道翻译式提示词改为情境式，让小孩在"具体场景"里挑回答。

| 原（翻译式） | 新（情景式） |
|---|---|
| 想邀请小伙伴一起玩： | 下课了，你跑到小伙伴跟前邀请他玩游戏，你说： |
| 在餐厅想点一份米饭： | 走进餐厅坐下，服务员问你想吃什么，你想点米饭： |
| 想告诉妈妈你在学校： | 妈妈打电话问你在哪儿，你回答： |
| …等共 51 题 | … |

**收益**：从"翻译题"变成"对话角色扮演"，更贴合 200 句日常口语的练习目标。

### 3. 英语难度递进（`index.html`）

`pickQuestionsSmart` 随机抽题后，按 tag 难度排序：

```js
const ENGLISH_DIFFICULTY = {
    '问候': 1, '感谢': 1, '道别': 1,
    '介绍': 2, '日常': 2,
    '游戏': 3, '饮食': 3, '学校': 3, '请求': 3,
    '情绪': 4,
};
```

每组 10 题从最易（问候）到最难（情绪）递进，避免"开局就难"的挫败感。

**注意**：仅对英语做排序，阅读维持随机（成语故事/生活常识 tag 数量少，难度差不明显）。

### 4. 结果页折叠正确题（`index.html`）

旧：10 题不分类，按 quiz 顺序依次展示 ✗/✓。
新：错题在上方独立展示，正确题折叠成 "✓ N 题全对 ▼" 摘要，点击展开。

**特例**：全对时不折叠，直接展开所有 10 行 ✓——保留小孩答对一组的成就感。

### 5. Review 修复（5 项）

代码评审发现的次生问题，全部在浏览器实测验证：

| # | 问题 | 修复 |
|---|---|---|
| #1 | 全对时折叠条隐藏所有 ✓，仪式感丢失 | `if (allCorrect)` 时直接展开 |
| #2 | wrong item 后接 summary 时 `:last-child` 失效，多出 1px 灰线 | `.detail-item.wrong { border-bottom: none }` |
| #3 | `animation-delay` 用 quiz 原始 index，首条 wrong 可能延迟 150ms+ 才出现 | 引入 `renderIdx` 参数，按渲染顺序级联 |
| #4 | `.detail-summary` 没有 inline delay，比上方 wrong 更早出现，视觉反向 | summary 加 `animation-delay: ${summaryDelay}s` |
| #5 | 未注册 tag fallback 到难度 2，无任何提示 | fallback 改为 99（排末尾），并通过 `warnedEnglishTags` 一次性 `console.warn` |
| #6 | `.expanded` 切换与全局 `.hidden` 工具类不一致 | 改用 `classList.toggle('hidden')` |
