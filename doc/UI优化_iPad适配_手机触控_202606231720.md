# UI 优化：iPad 适配 + 手机触控体验提升

提交：`3653810` (推送到 origin/main)
日期：2026-06-23

## 背景

主要使用端是手机和 iPad。原始 UI 只有一个 `@media (max-width: 480px)` 断点，且 `.app { max-width: 480px }` 写死——iPad 上整个应用被挤在 480px 窄栏里，两侧大量留白，按钮/字号都按手机比例。同时手机端部分触控目标小于 44pt，阅读 passage 内嵌滚动导致"小窗口里滚两次"的碎片体验。

## 改动详情（`index.html`）

### 1. 手机端触控区到 44pt

| 元素 | 原值 | 新值 |
|---|---|---|
| `.chip` | 无 min-height | `min-height: 44px` |
| `.choice-btn` | 无 min-height | `min-height: 52px` |

**收益**：符合 Apple Human Interface Guidelines 44pt 触控规范，减少误触。

### 2. 阅读 passage 放开高度

```css
/* 旧 */
.reading-passage {
    max-height: min(40vh, 240px);
    overflow-y: auto;
    -webkit-overflow-scrolling: touch;
}

/* 新 */
.reading-passage {
    /* 取消高度限制和内嵌滚动 */
}
```

**收益**：短文一眼看完，选项跟着整页滚动，不再"小窗口滚两次"。

### 3. iPad 断点 `@media (min-width: 768px)`

新增完整的平板适配块（约 50 行 CSS）：

| 区域 | 改动 |
|---|---|
| 整体布局 | `body` padding 加大、`.app` max-width 480 → 640、`.screen` padding 加大 + 圆角 28 → 32 |
| 首页 | mascot 3.5rem → 4.25rem、标题/科目卡 emoji/名称/描述全部上浮 |
| 设置页 | `.chip` 字号/min-height 上浮 |
| 答题页 | 题目数字 11vw → 8vw（避免太大）、`.choice-btn` min-height 52 → 60、numpad 按键 52 → 64px、numpad max-width 280 → 360 |
| 结果页 | 分数 2.25 → 2.75rem、详情列表取消 max-height |
| 弹窗 | wheel-modal max-width 340 → 420 |
| 通用按钮 | `.btn` min-height 50 → 56px、最大宽度上浮到 340 |

**收益**：iPad 上从"挤在手机栏里"变成"为 iPad 设计"——大触控目标、清晰字号、整体居中。

## 验证

**Chrome DevTools 三视口实测：**

- **iPhone 12 (390×844)**：首页/口算/阅读三页全部正常，passage 不再内嵌滚动
- **iPad mini (768×1024)**：app 加宽居中、numpad 按键放大、阅读一屏放下
- **桌面 (1440×900)**：命中 iPad 断点，app 居中且左右对称（width 640 / left 400 / right 1040）
